# trt-clinics

A public-facing directory of TRT and HRT clinics across the United States. Astro + Tailwind v4, data sourced from Google Places.

## Layout

```
src/
  pages/
    index.astro                              # national landing
    clinics/index.astro                      # all states
    clinics/[state]/index.astro              # cities in a state
    clinics/[state]/[city]/index.astro       # clinics in a city
    clinics/[state]/[city]/[clinic].astro    # clinic detail page
    telehealth/index.astro                   # placeholder — build out later
    about.astro, 404.astro
  layouts/BaseLayout.astro
  components/{StateGrid,ClinicCard}.astro
  data/{states.js, clinics.js}               # loads data/clinics.min.json at build
data/clinics.min.json                 # the dataset; regenerated by scripts
scripts/scrape_places.py                     # Google Places Text Search discovery
data/{seed-queries,seed-metros}.json         # search terms + metros per state
```

## Build & run

```bash
npm install
npm run dev        # http://localhost:4321
npm run build
```

## Data pipeline

All clinic data lives in `data/clinics.min.json`. Regenerate it with:

```bash
# pilot — four largest states
python3 scripts/scrape_places.py --states TX,CA,FL,NY --max-pages 2

# single metro smoke test
python3 scripts/scrape_places.py --states TX --metros "Dallas" --queries "trt clinic" --max-pages 1 --dry-run

# full sweep (all 50 states)
python3 scripts/scrape_places.py
```

The script calls Google Places (New) API Text Search once per (metro × query × page). Results are deduped by Places `id` and merged into the existing file. Set `GOOGLE_PLACES_API_KEY` in the env to override the default key.

## Deploying to Cloudflare Pages

Same pattern as roostmode / peptide-site / accessible-america — deploys via the Cloudflare Pages GitHub integration (no `wrangler.toml`, no CLI required):

1. Push this repo to GitHub
2. Cloudflare Pages dashboard → **Create project** → **Connect to Git** → pick the repo
3. Framework preset: **Astro**
4. Build command: `npm run build`
5. Build output directory: `dist`
6. Root directory: (leave blank unless repo is nested)
7. Environment variables: none required at build time — the dataset is committed as `data/clinics.min.json`, so builds are hermetic
8. Add the production domain under **Custom domains** once DNS is ready

Security headers and cache rules live in `public/_headers` — Cloudflare Pages picks them up automatically at deploy.

**Regenerating data:** runs locally (needs the Places API key). Commit the updated `data/clinics.min.json` and CF Pages rebuilds on push.

## Clinic record schema

Documented inline in `src/data/clinics.js`. Key fields: `placeId`, `slug`, `name`, `address`, `city`, `citySlug`, `state`, `stateSlug`, `phone`, `website`, `rating`, `ratingCount`, `priceLevel`, `googleUrl`, `hours[]`, `services[]`, `types[]`, `verified`, `featured`, `telehealth`, `source`, `lastSeenAt`.

Telehealth-only clinics (`telehealth: true`) are excluded from the state directory and will get their own section.
